# 2017A_FE-A_29

Which of the following can change the deadlock state of the transaction back to the normal state?

a) Commit
b) Drop
c) Rollback
d) Roll Forward
?
c) Rollback
### Explanation

When a **deadlock** occurs, the DBMS selects one transaction as a **victim** and aborts it. The aborted transaction's changes are undone using **ROLLBACK**, returning the database to its state before the transaction began so the other transaction can proceed normally.

The other options serve different purposes:
- **Commit** → finalizes a completed transaction
- **Drop** → removes database objects
- **Roll Forward** → re-applies changes during crash recovery (the opposite of rollback)
