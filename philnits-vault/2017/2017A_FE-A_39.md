# 2017A_FE-A_39

When the process shown in the procedure below is performed, in addition to the detection of message falsification, which of the following is what recipient B is able to do? [Procedure: (1) A hash function generates a digest from the message. (2) The digest is encrypted with sender A's private key to create a digital signature. (3) The message and signature are sent to recipient B. (4) B generates a digest from the received message using the same hash function. (5) B decrypts the signature using sender A's public key and compares the two digests.]

a) Confirming that the message is from sender A
b) Detecting wiretapping on the message
c) Identifying the areas of falsification
d) Preventing leakage of the message
?
a) Confirming that the message is from sender A
### Explanation

A digital signature created with **sender A's private key** can only be verified with **A's public key**. Since only A possesses the private key, a valid signature proves the message **originated from A** — providing **authentication and non-repudiation**.

What this does NOT provide:
- It does **not** prevent wiretapping (the message itself is not encrypted)
- It does **not** identify *which* part was altered — only that falsification occurred
- It does **not** prevent leakage (no confidentiality)
