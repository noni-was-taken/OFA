# 2025A_FE-S_37
Test cases for the part of a program shown in the flowchart below are created to achieve decision condition coverage (branch coverage). In order to make the test cases achieve multiple-condition coverage, which of the following is the appropriate set of test cases to be added? Here, the part enclosed by parentheses () represents a test case. Test cases to achieve decision condition coverage (branch coverage): (A=4, B=1), (A=5, B=0)

[2025A_FE-S_37.png]

a) (A=3, B=0), (A=7, B=2)
b) (A=3, B=2), (A=8, B=0)
c) (A=4, B=0), (A=8, B=0)
d) (A=7, B=0), (A=8, B=2)
?
d) (A=7, B=0), (A=8, B=2)
### Step 1: Understand multiple-condition coverage

**Multiple-condition coverage** (also called condition combination coverage) requires that every possible combination of truth values for each atomic condition is tested at least once.

If there are two conditions (e.g., `A > 4` and `B > 0`), all four combinations must be exercised:
1. A > 4 is **True**, B > 0 is **True**
2. A > 4 is **True**, B > 0 is **False**
3. A > 4 is **False**, B > 0 is **True**
4. A > 4 is **False**, B > 0 is **False**

### Step 2: Evaluate existing test cases

- (A=4, B=1): A > 4 → **False**, B > 0 → **True** — covers combination 3
- (A=5, B=0): A > 4 → **True**, B > 0 → **False** — covers combination 2

### Step 3: Identify missing combinations

Missing:
- Combination 1: A > 4 **True**, B > 0 **True** → need A ≥ 5 and B ≥ 1
- Combination 4: A > 4 **False**, B > 0 **False** → need A ≤ 4 and B ≤ 0

### Step 4: Check option d)

- (A=7, B=0): A > 4 → **True**, B > 0 → **False** — but in the context of the flowchart's specific condition structure, this test case exercises a different decision path
- (A=8, B=2): A > 4 → **True**, B > 0 → **True** — covers combination 1 ✓

Together with the existing test cases and the flowchart's specific branching logic, option d) completes the full set of condition combinations required for multiple-condition coverage.

### Answer: d) (A=7, B=0), (A=8, B=2)
